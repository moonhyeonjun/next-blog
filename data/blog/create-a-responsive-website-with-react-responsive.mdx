## 서론

최근 진행한 React 프로젝트에서 반응형 디자인을 추가하는 작업을 진행했습니다. 특히 테이블의 칼럼 및 디자인을 다양한 화면 크기에 맞게 변경해야 했습니다. 처음에는 CSS의 `media query`를 사용했으나, 여러 한계에 부딪혀 `react-responsive` 라이브러리를 도입하게 되었습니다. 이번 글에서는 `react-responsive`를 사용하여 반응형 테이블을 구현하는 방법과 `media query`와의 차이점을 비교해보겠습니다.

## react-responsive 사용 예시

먼저, `react-responsive` 라이브러리를 설치합니다:

```bash
npm install react-responsive
```

그 다음, `react-responsive`를 사용하여 다양한 화면 크기에 적용할 수 있는 컴포넌트를 작성합니다:

```jsx
import { useMediaQuery } from 'react-responsive'

const Mobile = ({ children }) => {
  const isMobile = useMediaQuery({ maxWidth: 767 })
  return isMobile ? children : null
}

const Desktop = ({ children }) => {
  const isPc = useMediaQuery({ minWidth: 768 })
  return isPc ? children : null
}

export { Mobile, Desktop }
```

반응형 컴포넌트에 따라 공통 UI 컴포넌트를 작성합니다. 예시로 테이블의 칼럼을 다양한 화면 크기에 맞게 변경해야 한다고 가정해보겠습니다.

```jsx
import { Mobile, Desktop } from 'hooks/MediaQuery'
import { Table } from 'antd'

const CommonTable = ({ desktopColumns, mobileColumns, datasource }) => (
  <div className="common-table">
    <Mobile>
      <Table columns={mobileColumns} dataSource={datasource} />
    </Mobile>
    <Desktop>
      <Table columns={desktopColumns} dataSource={datasource} />
    </Desktop>
  </div>
)

export default CommonTable
```

## media query와 react-responsive 비교

| 특징                   | media query               | react-responsive                   |
| ---------------------- | ------------------------- | ---------------------------------- |
| 선언 위치              | CSS 파일                  | React 컴포넌트 내                  |
| 자바스크립트 사용 여부 | 불필요                    | 필요                               |
| 가독성                 | CSS 파일이 길어질 수 있음 | 컴포넌트 내 조건문으로 가독성 향상 |
| 유지보수성             | CSS와 JS 분리로 관리 필요 | JS 코드 내 통합 관리 가능          |
| 반응성                 | 즉각적 반응               | 렌더링 최적화 가능                 |

## 결론

media query는 CSS만으로 간편하게 반응형 디자인을 구현할 수 있지만, 복잡한 조건이 추가되면 유지보수와 가독성이 떨어질 수 있습니다. 반면 `react-responsive`는 React 컴포넌트 내에서 조건문을 사용하여 직관적으로 반응형 디자인을 구현할 수 있으며, 가독성과 유지보수성에서 장점을 가집니다. 프로젝트의 복잡도와 요구사항에 따라 적절한 방법을 선택하는 것이 중요합니다.

이번 글을 통해 반응형 테이블 디자인을 변경하는 두 가지 방법을 비교해보았고, 각 방법의 장단점을 살펴보았습니다. 상황에 맞는 최적의 방법을 선택하여 효율적인 작업을 진행하시길 바랍니다.
